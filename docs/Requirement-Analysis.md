## Objective

The purpose of this document is to identify and document the functional requirements of the public SauceDemo application based on observable application behavior.

---

## Functional Requirements

| Requirement ID | Requirement Description                                                                  | Priority |
| -------------- | ---------------------------------------------------------------------------------------- | -------- |
| FR-001         | The application shall allow registered users to log in using valid credentials.          | High     |
| FR-002         | The application shall display an inventory of available products after successful login. | High     |
| FR-003         | The application shall allow users to sort products using the available sorting options.  | Medium   |
| FR-004         | The application shall allow users to view product details.                               | High     |
| FR-005         | The application shall allow users to add products to the shopping cart.                  | High     |
| FR-006         | The application shall allow users to remove products from the shopping cart.             | High     |
| FR-007         | The application shall allow users to view the shopping cart contents.                    | High     |
| FR-008         | The application shall allow users to proceed through the checkout process.               | High     |
| FR-009         | The application shall collect customer information during checkout.                      | High     |
| FR-010         | The application shall display an order overview before order confirmation.               | High     |
| FR-011         | The application shall complete the order after user confirmation.                        | High     |
| FR-012         | The application shall allow users to log out using the navigation menu.                  | High     |
